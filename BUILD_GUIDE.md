# دليل بناء وتشغيل تطبيق "غزة كاش" المتكامل (Gaza Cash ERP) 🚀

مرحباً بك في دليل التطوير والتعبئة الخاص بنظام **غزة كاش الذكي ERP**. 
تم تصميم وتطوير هذا النظام ليعمل كتطبيق سطح مكتب متكامل وقوي باستخدام **Electron + React + Vite + TypeScript**.

هذا الدليل يشرح بالتفصيل كيفية تشغيل التطبيق في بيئة التطوير، وكيفية تصديره وتعبئته إلى ملف تنفيذي مستقل (`.exe`) يعمل مباشرة على نظام التشغيل Windows دون الحاجة لتثبيت أي برمجيات مساعدة من قِبل المستخدم النهائي.

---

## 📌 المتطلبات الأساسية للتشغيل والبناء

قبل البدء، تأكد من تثبيت البرامج التالية على جهاز المطور:
1. **Node.js** (إصدار 18 أو أحدث - يوصى بإصدار LTS المستقر).
2. **مدير الحزم npm** (يأتي مدمجاً تلقائياً مع تثبيت Node.js).

---

## 🛠️ أولاً: أوامر التشغيل في بيئة التطوير (Development)

لتشغيل التطبيق وتجربة التعديلات بشكل مباشر مع ميزة التحديث الفوري:

1. **تثبيت الاعتمادات والاعتماديات** (في حال لم تقم بتثبيتها مسبقاً):
   ```bash
   npm install
   ```

2. **تشغيل خادم التطوير** (لتجربة التطبيق داخل متصفح الويب أو بيئة التطوير):
   ```bash
   npm run dev
   ```

3. **تشغيل التطبيق في بيئة Electron** (كـ تطبيق سطح مكتب):
   ```bash
   npm run electron:dev
   ```

---

## 📦 ثانياً: أوامر البناء وتوليد الملفات التنفيذية (`.exe`)

لقد قمنا بتهيئة وتثبيت أداة **`electron-builder`** وضبط إعداداتها بالكامل داخل ملف `package.json` مع أتمتة العملية لتتم بضغطة زر واحدة. 

تنتج عملية البناء نوعين أساسيين من الملفات التنفيذية لتناسب كافة رغبات عملائك:
1. **النسخة المحمولة (Portable)**: ملف واحد خفيف يعمل بمجرد النقر المزدوج عليه دون الحاجة لأي تثبيت (مثالي للتشغيل السريع من الفلاشة).
2. **نسخة التثبيت الاحترافية (NSIS Installer)**: معالج تثبيت يتيح للمستخدم تحديد مسار التثبيت، ويقوم بإنشاء اختصار للتطبيق على سطح المكتب وقائمة ابدأ، ويوفر خياراً سهلاً لإلغاء التثبيت.

### الأوامر المؤتمتة للبناء:

* **توليد كلا الإصدارين (المحمول والتثبيت) معاً في خطوة واحدة**:
  ```bash
  npm run build:win
  ```
  *(يقوم هذا الأمر ببناء واجهة React أولاً عبر Vite، ثم يشغّل معالج Electron Builder لتجميع إصدار الويندوز).*

* **توليد النسخة المحمولة (Portable) فقط**:
  ```bash
  npm run electron:build:portable
  ```

* **توليد نسخة التثبيت (NSIS Installer) فقط**:
  ```bash
  npm run electron:build:nsis
  ```

## 📂 ثالثاً: أين تجد الملفات الناتجة؟

بمجرد اكتمال عملية البناء بنجاح، سيقوم التطبيق بإنشاء مجلد جديد في جذر المشروع يسمى:
📂 **`release/`**

ستجد داخل هذا المجلد الملفات التالية جاهزة للنشر والتوزيع:
* **`غزة كاش [الاصدار] portable.exe`** 👈 هذا هو التطبيق المحمول الجاهز للتشغيل مباشرة.
* **`غزة كاش [الاصدار] nsis.exe`** 👈 هذا هو ملف التثبيت المخصص للعملاء لتثبيت البرنامج على أجهزتهم بشكل دائم.

---

## 🎨 رابعاً: ميزات إضافية تم دمجها وتكوينها

1. **أيقونة التطبيق الرسمية عالية الدقة (`icon.ico`)**:
   تم توليد أيقونة ويندوز مخصصة وعالية الدقة في المجلد `build/icon.ico` تحتوي على أحجام متعددة ومتوافقة (`256x256`, `128x128`, `64x64`, `32x32`, `16x16`) لضمان ظهور شعار التطبيق بشكل حاد واحترافي في:
   * شريط المهام (Taskbar).
   * سطح المكتب كاختصار.
   * قائمة ابدأ (Start Menu).
   * معالج التثبيت وإلغاء التثبيت.

2. **تجربة تثبيت مرنة (NSIS Configuration)**:
   تم إعداد ملف التثبيت بحيث:
   * يتيح للمستخدم اختيار تثبيت البرنامج للمستخدم الحالي أو لجميع مستخدمي الكمبيوتر (`perMachine: true`).
   * يسمح للمستخدم بتغيير مسار التثبيت الافتراضي واختيار القرص الصلب المناسب له (`allowToChangeInstallationDirectory: true`).
   * يظهر بشكل تفاعلي غير صامت لمنح العميل ثقة أكبر أثناء التثبيت الرسومي (`oneClick: false`).

## 🔍 خامساً: نصائح لضمان جودة واستقرار البناء

* **تحديث رقم الإصدار**: قبل تعبئة أي تحديث جديد وتوزيعه للعملاء، احرص على تعديل حقل `"version"` داخل ملف `package.json` (مثلاً من `1.1.0` إلى `1.2.0`) لكي يتغير اسم الملف المخرَج تلقائياً ويعكس التحديث الجديد.
* **أمن البيانات المحمية**: عند بناء النسخة النهائية للنشر، تأكد من إعداد متغيرات البيئة الحساسة إن وُجدت، أو تركها لقاعدة البيانات المحلية المؤمنة لتجنب كشف أي مفاتيح برمجية.
